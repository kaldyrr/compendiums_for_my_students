# 📋 Описание задания

## Актуализация 2026-06

- Используйте актуальный Git 2.5x с Git Credential Manager.
- Для GitHub рекомендуются SSH-ключи Ed25519 или fine-grained PAT для HTTPS.
- В будущей интерактивной версии урок проверяет `git --version`, `git config --global --list`, наличие SSH-ключа и успешный `ssh -T git@github.com`.

## Цель: Научиться устанавливать и настраивать Git на локальной машине для дальнейшей работы с системой контроля версий.

## Задачи:

- Установить Git на вашу операционную систему

- Настроить базовую конфигурацию Git

- Проверить корректность установки

- Создать и настроить SSH-ключ для безопасного подключения к GitHub

# 🛠️ Пошаговое выполнение

## Шаг 1: Установка Git

### Для Windows:

Перейдите на [официальный сайт Git](https://git-scm.com/install/windows)

Скачайте установщик для Windows

Запустите установщик и следуйте инструкциям:

Выберите компоненты: отметьте "Git Bash Here", "Git GUI Here"

Выберите редактор по умолчанию: рекомендуется VS Code

Настройте имя основной ветки: выберите "main"

Настройте PATH: выберите "Git from the command line and also from 3rd-party software"

Выберите HTTPS транспорт: "Use the OpenSSL library"

Настройте окончания строк: "Checkout Windows-style, commit Unix-style line endings"

Настройте терминал: "Use MinTTY"

Выберите поведение `git pull`: осознанно выберите rebase или fast-forward only для учебной линейной истории; merge-поведение объясняется в уроке про ветки.

Выберите credential helper: "Git Credential Manager"

Включите файловый кэш: "Enable file system caching"


### Для Linux (Ubuntu/Debian):
```bash
sudo apt install git
```
### Для Linux (Fedora):
```bash
sudo dnf install git
```
### Для Linux (Arch, Manjaro):
```bash
sudo pacman -S git
```


### Для macOS:
```bash
# Способ 1: через Homebrew
brew install git

# Способ 2: через Xcode Command Line Tools
xcode-select --install
```
## Шаг 2: Базовая настройка Git

Откройте терминал (Git Bash на Windows) и выполните следующие команды:
```bash
# Настройка пользователя (замените на свои данные)
git config --global user.name "Ваше Имя"
git config --global user.email "your.email@example.com"

# Настройка редактора (если используете VS Code)
git config --global core.editor "code --wait"

# Установка основной ветки как main
git config --global init.defaultBranch main

# Настройка поведения git pull
git config --global pull.rebase true

# Включение цветного вывода
git config --global color.ui auto

# Настройка обработки окончаний строк (для Windows)
git config --global core.autocrlf true

# Настройка обработки окончаний строк (для Linux/macOS)
git config --global core.autocrlf input
```

## Шаг 3: Проверка установки
```bash
# Проверка версии Git
git --version

# Просмотр текущей конфигурации
git config --list

# Безопасная проверка только глобальных учебных настроек
git config --global --list

# Проверка конкретных настроек
git config user.name
git config user.email
```

## Шаг 4: Настройка SSH-ключа

Создание SSH-ключа:
```bash
# Создание нового SSH-ключа (замените email на свой)
ssh-keygen -t ed25519 -C "your.email@example.com"

# На вопрос "Enter file in which to save the key" нажмите Enter
# При запросе пароля можно оставить пустым или установить надежный пароль
```

Добавление ключа в SSH-агент:
```bash
# Запуск SSH-агента
eval "$(ssh-agent -s)"

# Добавление приватного ключа
ssh-add ~/.ssh/id_ed25519
```

Копирование публичного ключа:
```bash
# Просмотр и копирование публичного ключа
cat ~/.ssh/id_ed25519.pub

# Для Windows также можно использовать:
clip < ~/.ssh/id_ed25519.pub
```

## Шаг 5: Добавление SSH-ключа в GitHub

1. Перейдите на GitHub.com

2. В правом верхнем углу нажмите на аватар → Settings

3. В левом сайдбаре выберите SSH and GPG keys

4. Нажмите New SSH key

5. В поле "Title" введите описательное имя (например, "My Laptop")

6. В поле "Key" вставьте содержимое вашего публичного ключа

7. Нажмите Add SSH key

## Шаг 6: Проверка подключения
```bash
# Проверка подключения к GitHub
ssh -T git@github.com
```
> Вы должны увидеть сообщение: Hi username! You've successfully authenticated, but GitHub does not provide shell access.

# ✅ Критерии успешного выполнения

## 🐛 Возможные проблемы и решения

Проблема: Команда git не распознается в терминале
Решение: Перезапустите терминал или перезагрузите компьютер после установки

Проблема: Ошибка при создании SSH-ключа
Решение: Убедитесь, что папка ~/.ssh существует, или создайте ее вручную

Проблема: SSH-агент не запускается
Решение: Попробуйте команду eval $(ssh-agent) или перезапустите терминал

Проблема: Не удается подключиться к GitHub через SSH
Решение: Проверьте, что публичный ключ правильно скопирован в GitHub (без лишних пробелов или переносов)
